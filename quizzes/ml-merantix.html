Q01 Given the train and test loss curves above, what actions would help? ▾
✦ Correct Answers
✓ Add dropout
✓ Gather more training data
✗ Increase model capacity
✗ Debug network implementation

Explanation

The graph shows overfitting: train loss → ~0.27, test loss plateaus at ~0.5. The model memorizes training data but can't generalize.

Q02 What is the difference between a naive Bayes and a logistic regression model? ▾
✦ Correct Answers
✓ Naive Bayes assumes features are conditionally independent; logistic regression does not
✓ Naive Bayes models P(X|y); logistic regression models P(y|X)
✗ Naive Bayes is discriminative, logistic regression generative
✗ Logistic regression assumes Gaussian distribution

Explanation

Naive Bayes is a generative model — it models the joint distribution P(X,y) = P(X|y)·P(y), assuming all features are independent given the class. This "naive" assumption is the defining characteristic.

Logistic regression is discriminative — it directly models the posterior P(y|X) without making assumptions about the data distribution.

Q03 Why is a computation written in Numpy usually faster than the same computation implemented in native Python? ▾
✦ Correct Answers
✓ NumPy operations are vectorized (no Python loops)
✓ NumPy operations are parallelized over multiple CPUs if hardware supports it
✗ NumPy operations are parallelized on a GPU
✗ NumPy leverages just-in-time compilation

Explanation

Vectorization is the primary speedup: NumPy calls optimized C/Fortran routines that process entire arrays at once, bypassing Python's slow interpreter loop. Multi-CPU parallelism via BLAS/LAPACK adds further speedup for linear algebra.

GPU support belongs to CuPy/PyTorch. JIT compilation belongs to Numba/JAX, not NumPy.

Q05 As a rule of thumb, how does the accuracy of a model tend to scale with the amount of training data? ▾
✦ Correct Answer
✓ Logarithmically
✗ Linearly
✗ Exponentially
✗ Not related

Explanation

Empirically, accuracy gains diminish as data grows. Doubling data from 100→200 samples improves accuracy dramatically; doubling from 1M→2M barely changes it. This is a logarithmic relationship.

This is why data collection has diminishing returns, and why techniques like data augmentation and transfer learning exist.

Q06 What is the difference between variational inference and sampling in graphical models? ▾
✦ Correct Answers
✓ Sampling gives an approximate value of the true posterior
✓ Sampling is often much harder to implement
✗ Variational inference gives the true value of an approximate posterior
✗ Variational inference requires a variational autoencoder

Explanation

Variational Inference (VI): Turns Bayesian inference into an optimization problem. Finds the best approximating distribution q(z) ≈ p(z|x) from a family of distributions. Fast but biased (q is never the true posterior).

Sampling (MCMC): Draws samples that asymptotically converge to the true posterior. Unbiased in the limit, but requires careful tuning and can be slow. VAEs use VI but VI is a standalone framework.

Q07 You are given a revolver with six slots. There are two adjacent bullets. You have to shoot twice and are given the chance to rotate the cylinder randomly in-between. How do you maximize your chance of survival? ▾
✦ Correct Answer
✓ Do not rotate cylinder
✗ Rotate cylinder
✗ It does not matter

Explanation — Conditional Probability

Without rotating: You survived shot 1, so you know an empty chamber fired. With 2 adjacent bullets (say pos 1,2), firing positions 3,4,5,6 are empty. Next chamber after 3→4✓, 4→5✓, 5→6✓, 6→1✗. P(survive) = 3/4 = 75%

With rotating: P(survive) = 4/6 ≈ 67%

Surviving shot 1 gives you information — you're likely away from the bullet cluster. This is Bayesian reasoning.

Q08 While training a model, you see it start to output NaN as its loss value. Which of the following is most likely to remedy this? ▾
✦ Correct Answer
✓ Reduce your learning rate
✗ Reduce training data size
✗ Increase parameters
✗ Increase learning rate

Explanation

NaN loss is the signature of exploding gradients caused by too large a learning rate. The update step overshoots, parameter values blow up to ±∞, and the loss becomes NaN.

Fix: reduce LR (often by 10×), add gradient clipping ‖g‖ ≤ threshold, or use adaptive optimizers like Adam that scale LR per-parameter.

Q09 Why do researchers tend to split their dataset into 3 parts (training, validation, testing)? ▾
✦ Correct Answers
✓ To make sure the model generalizes well
✓ To be able to tune hyperparameters properly
✗ To have 2 independent subsets to constantly evaluate on
✗ There is no good reason — just convention

Explanation

Validation set: Used to tune hyperparameters (LR, architecture, regularization). Each tuning decision "uses up" this data's independence.

Test set: Touched exactly once at the very end for an unbiased estimate of real-world performance. Never used for tuning — that would cause optimistic bias.

The test set must remain completely blind; evaluating on it repeatedly invalidates it.

Q10 You are trying to predict a continuous variable with your model. Instead of training it for regression you decide to discretize the variable and train the model for classification instead. Why would you do it? ▾
✦ Correct Answer
✓ When training for regression the optimization can be less stable
✗ There is no argument for doing so
✗ The accuracy will be much higher
✗ You cannot train a neural network for regression

Explanation

MSE/MAE regression loss can have noisy, poorly-scaled gradients for large output ranges. Cross-entropy loss for classification often provides smoother gradient flow and more stable training.

This is a deliberate trade-off: you lose information precision but gain training stability. Neural networks absolutely can do regression (use linear output + MSE).

Q11 You see this training plot, the learning rate was constant during training. What do you suspect went wrong? ▾
✦ Correct Answer
✓ Learning rate too low
✗ Learning rate too high
✗ Bad model initialization
✗ Model capacity not large enough

Explanation

A very low LR means gradient steps are tiny — the model barely moves in parameter space. This creates a long flat plateau. Eventually the accumulated tiny steps find a descent direction, producing the late sharp drop.

High LR → erratic/diverging. Bad init → instability early on. Low capacity → permanent floor (no recovery). This pattern uniquely points to LR too small.

Q12 You're considering adding one of the following layers in the middle of your neural network. Which of them are NOT differentiable for backpropagation? ▾
✦ Correct Answer
✓ Argmax
✗ Batch normalization
✗ Convolution
✗ Dropout

Explanation

Argmax outputs a discrete integer index. An infinitesimal change in input doesn't smoothly change which index is returned — the gradient is either 0 or undefined. You can't backpropagate through it.

Solutions: use Softmax (soft approximation), Gumbel-Softmax (differentiable sampling), or straight-through estimator. Batch norm, convolution, and dropout are all differentiable.

Q13 How does the receptive field of three consecutive layers of stride-1 3x3 convolutions compare to that of a single layer of a 7x7 convolution? ▾
✦ Correct Answer
✓ They have the same receptive field (7×7)
✗ The single 7×7 has a larger receptive field
✗ The three 3×3 layers have a larger receptive field

Explanation

Each stride-1 3×3 conv adds 2 to the receptive field: 3 → 5 → 7. Three stacked 3×3s see a 7×7 patch of the original input, identical to a single 7×7 conv.

This insight drives architectures like VGGNet. Three 3×3s are preferred because they're cheaper (Q14) and have more non-linearities (ReLU between each).

Q14 How does the number of parameters in three consecutive layers of stride-1 3x3 convolutions compare to that of a single layer of a 7x7 convolution? ▾
✦ Correct Answer
✓ The three 3×3 layers have fewer parameters
✗ The single 7×7 has fewer parameters
✗ They have the same number

Explanation

One 7×7: 7 × 7 × C × C = 49C² Three 3×3: 3 × (3 × 3 × C × C) = 27C² Ratio: 27/49 ≈ 55% — saves ~45% of parameters!

Same receptive field, 45% fewer parameters, and three non-linearities instead of one. This is why VGG stacks 3×3s.

Q16 You're training a softmax classifier on 23 classes. What do you expect the cross-entropy loss and accuracy to be at the beginning, assuming proper initialization? ▾
✦ Correct Answer
✓ Loss: ln(23), accuracy: 1/23
✗ Loss: ln(22), accuracy: 1/22
✗ Loss: ln(23), accuracy: 1 − 1/23

Explanation

With proper init: softmax ≈ uniform → p(correct class) = 1/23 Cross-entropy loss = -log(1/23) = log(23) = ln(23) ≈ 3.14 Accuracy = 1/23 ≈ 4.35% (random chance)

This is a useful sanity check: if your initial loss ≠ ln(num_classes), something is wrong with initialization.

Q17 In the Scaled Dot-Product Attention mechanism, the dot products of the queries and keys are scaled by a factor of 1/sqrt(d_k) before applying the softmax function. What is the primary architectural justification for this specific scaling factor? ▾
✦ Correct Answer
✓ Counteracts dot products growing large with d_k, preventing vanishing softmax gradients
✗ Normalizes Q and K to unit norm
✗ Reduces complexity from quadratic to linear
✗ Ensures attention weights sum to 1

Explanation

If q,k ~ N(0,1) independently, then q·k ~ N(0, d_k) Std dev grows as √d_k → large inputs push softmax to near-0/1 (saturation) Dividing by √d_k → variance = 1 regardless of dimension → healthy gradients

Softmax already ensures weights sum to 1 regardless. Scaling doesn't change complexity — attention is O(n²) either way.

Q18 What can you try to increase the performance of a model when training and testing accuracy converges to about the same? ▾
✦ Correct Answer
✓ Increase the model capacity
✗ Decrease model capacity
✗ Performance cannot get any better
✗ Early stopping

Explanation — Underfitting

Train ≈ Test (both low) = underfitting. The model is too simple to learn the task. The cure is more capacity: more layers, more neurons, more complex architecture.

Bias-variance tradeoff: high bias (underfitting) needs more capacity. High variance (overfitting) needs regularization or more data.

Q19 You are training a model using ReLU activation functions. After some training, you notice many units never activate. What are some plausible actions you could take to get more units to activate? ▾
✦ Correct Answers
✓ Use Leaky ReLU units
✓ Use momentum-based gradient method
✓ Add positive bias when initializing weights
✗ Decrease batch size

Explanation

Leaky ReLU: f(x) = max(αx, x) where α≈0.01. Even dead neurons get a tiny gradient and can recover.

Momentum: Accumulated gradient velocity can push weights out of the zero-gradient region.

Positive bias init: Pre-activations start positive → ReLUs start active → gradients flow from step 1.

Q20 Let's compare the following methods for hyperparameter optimization: Random Search, Grid Search and Bayesian Hyperparameter Optimization. Which of the following statements are true? ▾
✦ Correct Answers
✓ Bayesian HPO is preferable — it leverages knowledge from prior runs
✓ Random Search is more efficient than Grid Search
✗ Grid Search is better because it's deterministic
✗ Bayesian and Random Search are equally preferable

Explanation

Grid Search wastes runs on unimportant hyperparameters. Random Search allocates coverage more efficiently to important dimensions (Bergstra & Bengio, 2012).

Bayesian HPO (e.g., Optuna, BoTorch) builds a surrogate model of the loss landscape and uses acquisition functions to intelligently pick the next trial — far more sample-efficient than either search method.

Q27 Which of the following are techniques for parallelizing neural net training? ▾
✦ Correct Answers
✓ Train model with different examples on different servers (data parallelism)
✓ Train parts of very large models on different servers (model parallelism)
✓ Keep input data and model parameters on separate servers (parameter server)
✗ Calculate forward and backward pass on separate servers

Explanation

Data parallelism: Replicate model on N GPUs, split mini-batch, average gradients. Most common approach (DDP in PyTorch).

Model parallelism: Split layers across devices. Used when model is too large for a single GPU (pipeline parallelism).

Parameter server: Workers compute gradients, push to a central server that updates and broadcasts parameters.

Q29 EM (Expectation Maximization) is: ▾
✦ Correct Answer
✓ An optimization technique for latent variable models
✗ A segmentation algorithm
✗ A clustering algorithm
✗ An inference algorithm for conditional random fields

Explanation

E-step: Compute expected values of latent variables: Q(θ|θ_old) = E[log P(X,Z|θ)]

M-step: Maximize Q w.r.t. θ to update parameters.

EM can be applied to clustering (e.g. GMMs), but that's a use case. It's a general-purpose latent variable optimization algorithm. It can be used for CRF inference but isn't specifically designed for it.

Q30 A class of functions, F, can shatter a dataset of size n if: ▾
✦ Correct Answer
✓ It achieves zero training loss on every possible labelling of the dataset
✗ It achieves zero evaluation loss on every labelling
✗ It has sufficient capacity
✗ It has infinite VC dimension

Explanation — VC Theory

Shattering means F can correctly classify all 2ⁿ possible binary labellings of n points. VC dimension = the largest n for which such shattering is possible. Infinite VC dimension means shattering any finite set — stronger than shattering one specific set of size n.

Q31 Starting from x=0, which of the following optimisation methods will converge to the minimum at x=2 of the function f(x) shown above? ▾
✦ Correct Answers
✓ Newton's method applied to f(x)
✓ Newton's method applied to df/dx
✓ Gradient descent with learning rate 1
✗ Gradient descent with learning rate 4

Explanation

For a quadratic f with f''≈0.5, GD update: x ← x - α·f'(x). Convergence requires α·f'' < 2, i.e. α < 4. LR=1 → 0.5 < 2 ✓. LR=4 → 4×0.5 = 2 → oscillation, no convergence ✗.

Newton's method (x ← x - f'/f'') solves a quadratic exactly in one step. Applied to the derivative (root-finding), it also finds the minimum exactly.

Q34 Sparse Mixture of Experts (MoE) models (like Mixtral) aim to decouple inference cost from model size. What is the role of the 'router' or 'gating network' in this architecture? ▾
✦ Correct Answer
✓ Dynamically selects top-k expert layers per token — only a fraction of parameters are active
✗ Averages all expert weights into one dense layer
✗ Routes tokens to GPUs for load balancing
✗ Decides whether to use external memory or internal memory

Explanation

In models like Mixtral 8×7B, each layer has 8 expert FFN networks. The router selects top-2 for each token. This means 2/8 = 25% of expert parameters are used per forward pass, decoupling inference FLOPS from total model size. The router is a learned linear layer.

Q35 Assuming your data exists on some compact subset of R^n, how many hidden layers are necessary for a neural network to be able to represent any continuous function? ▾
✦ Correct Answer
✓ 1
✗ 0
✗ 2
✗ 10

Explanation — Universal Approximation Theorem

The UAT (Cybenko 1989, Hornik 1991) proves that a single hidden layer with a non-linear activation and sufficiently many neurons can approximate any continuous function on a compact set to arbitrary precision.

In practice, deeper networks (more layers) learn more efficiently with fewer total neurons and generalize better.

Q36 You train a GAN to generate pictures of cute dogs and cats (30% dogs, 70% cats)... After training, you feed the 0 vector through your generator. Your sample looks nothing like a dog. Why? ▾
✦ Correct Answers
✓ The network has seen almost no training points "close" to the mode
✓ The mode corresponds to the most probable class (cat, not dog)
✗ The network learned cat-dog hybrids only
✗ Gaussian(0,1) is not the right distribution

Explanation

Concentration of measure: In high dimensions (e.g. 512-d latent space), the Gaussian's probability mass is concentrated in a thin shell far from the origin. The zero vector is essentially an outlier — the network has never "seen" it during training.

Class imbalance: With 70% cats, the latent space region around zero maps to the dominant class (cat), not dog.

Q37 When training a neural net, what is an advantage of using minibatch gradient descent over single-example gradient descent? ▾
✦ Correct Answers
✓ Smoother convergence
✓ Higher training throughput
✗ Reduces training loss
✗ Allows for batch normalization

Explanation

Smoother convergence: Averaging gradients over B examples reduces noise variance by ~1/B, giving more stable update directions.

Higher throughput: GPUs execute a batch of 32 in roughly the same wall time as 1 example due to parallelism — effectively 32× more data processed per unit time.

Batch norm requires batches as a technical prerequisite, but isn't a primary advantage of minibatch GD itself.

✗ Adding a negative multiple of the gradient

02 — LEARNING

Deep Dive Study Material

Overfitting, Underfitting & Regularization

The bias-variance tradeoff is the central challenge of ML. Understanding it explains half of all training decisions.

  • Overfitting (high variance): Train loss ≪ test loss. Model memorizes data. Fix: more data, dropout, L1/L2 regularization, early stopping, data augmentation.
  • Underfitting (high bias): Train ≈ test loss, both high. Model too simple. Fix: more layers, more neurons, fewer regularization constraints.
  • Dropout: During training, randomly zero each neuron with probability p. Forces network to learn redundant representations. At inference, scale by (1-p).
  • L2 regularization (weight decay): Add λ·‖w‖² to the loss. Penalizes large weights, encourages smoothness.
  • Early stopping: Monitor val loss; stop when it starts increasing.

Neural Network Fundamentals

Core building blocks every practitioner must internalize.

  • Universal Approximation Theorem: 1 hidden layer + enough neurons ≈ any continuous function. But deeper is more efficient in practice.
  • Backpropagation: Chain rule applied recursively. Compute ∂L/∂w for every weight by propagating errors backward.
  • Initialization: Xavier/Glorot for tanh, He/Kaiming for ReLU. Proper init → initial loss = ln(num_classes) for cross-entropy.
  • Vanishing/Exploding Gradients: Deep nets struggle with gradient flow. Solutions: residual connections (ResNet), LSTM gates, batch normalization, gradient clipping.
  • Activation functions: ReLU (fast, can die), Leaky ReLU (no dying), GELU (smooth, used in transformers), Sigmoid/Tanh (saturate, avoid in hidden layers).
UAT backprop chain rule He init ReLU skip connections
Convolutional Neural Networks
  • Output size formula: ⌊(W - K + 2P) / S⌋ + 1, where W=input, K=kernel, P=padding, S=stride.
  • Receptive field: N stacked stride-1 K×K convolutions → (N(K-1)+1)² receptive field. Three 3×3s = one 7×7.
  • Parameter efficiency: Three 3×3s (27C²) vs one 7×7 (49C²) for same receptive field — 45% fewer params, 3× non-linearities.
  • Key architectures: LeNet → AlexNet → VGGNet → GoogLeNet → ResNet → DenseNet → EfficientNet.
  • ResNet skip connections: y = F(x) + x. If F(x) ≈ 0, the layer becomes identity → easy gradient flow for very deep nets.
Conv output: floor((W - K + 2P) / S) + 1 Receptive field: 1 + N*(K-1) for N stride-1 K×K convs

Optimizers & Training Dynamics
  • SGD: w ← w - α·∇L. Simple, needs tuning. Works well with momentum.
  • Adam: Adaptive per-parameter LR. Combines momentum (1st moment) and RMSprop (2nd moment). Default for most tasks.
  • RMSprop: Divides LR by moving average of squared gradients. An optimizer, NOT a regularizer.
  • Learning rate too high → NaN loss (exploding gradients), oscillation.
  • Learning rate too low → flat loss curve, extremely slow convergence.
  • LR schedulers: Cosine annealing, step decay, warmup. Crucial for transformers.
  • Gradient clipping: Clip ‖g‖ to a threshold. Standard practice for RNNs and transformers to prevent NaN.

Transformers & Attention
  • Scaled Dot-Product Attention: Attention(Q,K,V) = softmax(QKᵀ/√d_k)·V
  • Why √d_k scaling? Without it, dot products grow with d_k → softmax saturates → vanishing gradients.
  • Multi-head attention: Run h attention heads in parallel with different learned projections, concatenate.
  • Complexity: O(n²·d) where n=sequence length. Quadratic in sequence length.
  • Sparse MoE: Replace dense FFN with N expert FFNs + router. Router selects top-k experts per token. Decouples model size from inference cost.
  • Position encoding: Transformers have no recurrence → must inject position info (sinusoidal or learned embeddings).
Attention(Q,K,V) = softmax(QKᵀ / √d_k) · V d_k = dimension of keys/queries Scaling keeps variance ≈ 1 regardless of d_k

Probabilistic ML & Bayesian Methods
  • Generative vs Discriminative: Generative (Naive Bayes, GMMs) models P(X,y); discriminative (logistic regression, SVMs) models P(y|X) directly.
  • Naive Bayes: P(y|X) ∝ P(y)·∏P(xᵢ|y). "Naive" = features conditionally independent given class. Simple, interpretable, fast.
  • EM Algorithm: E-step computes expected latent variables; M-step maximizes expected complete-data log-likelihood. Used for GMMs, HMMs, k-means (special case).
  • Variational Inference: Approximate true posterior p(z|x) with a simpler distribution q(z). Maximize ELBO = E[log p(x,z)] - E[log q(z)]. VAEs use VI.
  • MCMC Sampling: Draw samples from a Markov chain whose stationary distribution is the target posterior. Asymptotically exact but computationally expensive.
Bayes theorem posterior ELBO MCMC EM algorithm
Recurrent Networks & LSTMs
  • Vanilla RNN: hₜ = tanh(Wₕhₜ₋₁ + Wₓxₜ + b). Suffers vanishing/exploding gradients over long sequences. Bad at long-term dependencies.
  • LSTM: Adds cell state (conveyor belt) and three gates: forget (f), input (i), output (o). Gates are sigmoid → control information flow.
  • GRU: Simplified LSTM with only 2 gates (reset, update). Couples input and forget: update = 1 - reset. More parameter-efficient.
  • Key difference from LSTM: GRU couples gates; LSTM has independent gates. Both are types of RNN.
LSTM gates: f = σ(Wf·[h,x] + bf) ← forget i = σ(Wi·[h,x] + bi) ← input o = σ(Wo·[h,x] + bo) ← output C = f⊙C_prev + i⊙tanh(Wc·[h,x]+bc)
LSTM GRU vanishing gradient cell state gating
Adversarial ML & Model Evaluation
  • FGSM: x_adv = x + ε·sign(∇ₓL). Minimal perturbation, maximal loss increase. Invented by Goodfellow et al. 2014.
  • ROC curve: TPR vs FPR. AUC-ROC robust to class imbalance. Higher AUC = better.
  • PR curve: Precision vs Recall. Better metric for imbalanced datasets with rare positives. AUC-PR is more informative than AUC-ROC for severe imbalance.
  • VC Dimension: Measures classifier capacity. A linear classifier in d dimensions has VC dim = d+1. DNNs have very high (sometimes infinite) VC dim.
  • Shattering: F shatters n points if it can achieve any labeling. VC dim = max n such that F shatters some n points.

NumPy & Python Fundamentals
  • Why NumPy is fast: Vectorized C operations (no Python loop overhead), BLAS/LAPACK for linear algebra, optional SIMD/multi-core parallelism.
  • Broadcasting: Operations on arrays of different shapes — NumPy automatically expands dimensions. Eliminates explicit loops.
  • Dict keys: Must be hashable (immutable). str, int, float, tuple ✓. list, dict, set ✗.
  • JIT vs vectorization: Numba/JAX provide JIT compilation. NumPy uses pre-compiled C, not JIT.
  • GPU arrays: CuPy (NumPy API on GPU), PyTorch tensors. NumPy itself is CPU-only.

Study to understanding on multi-select topics. The multi-select questions penalise half-knowledge — you need to know not just the correct answers but why the wrong options are wrong:

  • Generative vs discriminative: Naive Bayes models P(X|y) and requires the full joint; logistic regression models P(y|X) directly — subtle but testable
  • VI vs MCMC: Variational inference is biased but tractable (optimisation); MCMC is asymptotically exact but hard to tune (sampling)
  • Regularisation vs optimisation: RMSprop is an optimizer — it adapts learning rates, does not penalise model complexity
  • ROC vs PR curves: ROC is invariant to class imbalance; PR degrades when negatives dominate — know which to use and why
  • LSTM vs GRU: LSTM has independent forget/input/output gates; GRU couples them (update = 1 − reset). Both are types of RNN
